# biPlanes 详解
BMP信息头里的“颜色平面数”（biPlanes）是一个很容易让人困惑，但实际上又非常简单的概念。简单来说，**它是一个历史遗留字段，在现在的BMP图片里，它必须永远等于1，没有例外**。

这就好比汽车上的备用轮胎架。以前的车真的需要把备胎挂在外面，现在的车很多已经不用了，但车身的设计还是保留了那个位置（或者一个盖子），用来保持车身造型的统一。`biPlanes`就是BMP格式为了保持结构统一而留下的“备胎架”。

---

### 1. “颜色平面”到底是什么？

为了理解它，我们先回到计算机图形学的早期（上世纪80、90年代）。那时的显示设备和现在的思路不太一样。

你可以把“颜色平面”想象成一张透明的塑料片。

-   **早期的一些显卡和图像格式**，会把一张彩色图片拆分成几张单色的“透明塑料片”。比如，一张图片可以拆成三个平面：一个红色平面、一个绿色平面、一个蓝色平面。每个平面只存储自己对应颜色的亮度信息。
-   显示图片时，显卡会把这几个平面像叠幻灯片一样叠在一起，最终合成一张完整的彩色图像。
-   在这种“多平面”模式下，**“颜色平面数”就是指这些透明塑料片的数量**。

### 2. 为什么BMP的“颜色平面数”是1？

明白了上面的概念，就很容易理解BMP的做法了。

-   **BMP格式从一出生，就没有采用这种“多平面”的存储方式。** 它采用的是另一种更直接、也一直沿用至今的方法：**“像素合一”**。
-   在BMP文件里，所有关于一个像素的颜色信息（比如蓝色、绿色、红色的亮度）是**紧挨着连续存储**的。它没有把颜色拆分成独立的平面，只有一个“平面”，这个平面里包含了所有颜色信息。
-   所以，在BMP的信息头里，`biPlanes`这个字段就必须设置为**1**，用来告诉读取文件的软件：“我这个图片是单平面、像素连续存储的格式”。

### 3. 为什么这个字段没有被去掉？

这是为了格式的**向前兼容**。BMP文件格式在设计之初就定义好了这个结构。后来即使技术发展，这个字段变成了固定的“1”，也不能把它删除，否则所有旧的软件就不知道如何读取新的文件了。它必须永远待在原来的位置，保持整个信息头结构的长度不变。这和上一轮我们聊的“保留字段”的用意是类似的。

### 4. 一个小类比：更像RGB通道？

虽然“颜色平面”的概念在BMP里不适用，但你可能听过另一个词：**RGB通道**。

-   **通道**（如Photoshop里的红、绿、蓝通道）指的是**逻辑上**对颜色分量的分离，它们可以叠加成彩色图像。
-   **BMP的“平面”** 指的是**物理上**（在文件里）的存储结构。BMP选择了把所有通道的数据混在一起存，所以它只有一个物理“平面”。

所以，你可以这样记：BMP文件把所有的颜色信息都堆在了这一个“平面”里，而不是分别放在几个不同的“平面”里。因此，`biPlanes = 1` 就是告诉我们这个事实。
