# fwrite 函数

如果说 `fread` 是从文件**读取**数据到内存，那么 `fwrite` 就是**把内存中的数据写入文件**。它和 `fread` 是一对好搭档，经常一起使用。

---

## 1. `fwrite` 长什么样？（函数原型）

```c
size_t fwrite(const void *ptr, size_t size, size_t nmemb, FILE *stream);
```

参数解释：

- **`ptr`**：一个指针，指向你要写入文件的数据在内存中的位置。你可以把它想象成“数据源头”。
- **`size`**：每个要写入的数据块的大小（单位是字节）。比如你要写入一个整数，一般占 4 字节，这里就填 `sizeof(int)`。
- **`nmemb`**：要写入多少个这样大小的数据块。比如你要写 10 个整数，这里就填 `10`。
- **`stream`**：文件指针，就是你用 `fopen` 打开文件后得到的，它告诉 `fwrite` 要写入哪个文件。

**返回值**：返回实际成功写入的数据块个数（就是 `nmemb` 那个单位）。正常情况下应该等于你请求的 `nmemb`，如果小于 `nmemb`，说明发生了错误（比如磁盘满了、权限问题等）。

---

## 2. 举个最简单的例子：把一个整数写入文件

假设我们要把整数 `12345` 写入一个二进制文件 `data.bin`。

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("data.bin", "wb");  // 以二进制写方式打开（如果文件不存在会创建）
    if (fp == NULL) {
        printf("文件打开失败！\n");
        return 1;
    }

    int num = 12345;
    size_t result = fwrite(&num, sizeof(int), 1, fp);  // 写 1 个整数

    if (result != 1) {
        printf("写入失败\n");
    } else {
        printf("写入成功\n");
    }

    fclose(fp);
    return 0;
}
```

这里：
- `&num` 是数据来源地址。
- `sizeof(int)` 是每个数据块的大小。
- `1` 是要写入的数据块个数。
- `fp` 是文件指针。

`fwrite` 会尝试把 `sizeof(int)` 个字节从 `&num` 复制到文件中。

---

## 3. 写多个数据（比如数组）

假设你有一个整数数组，想一次性写入文件：

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("data.bin", "wb");
    if (fp == NULL) return 1;

    int arr[5] = {10, 20, 30, 40, 50};
    size_t count = fwrite(arr, sizeof(int), 5, fp);

    if (count == 5) {
        printf("成功写入 5 个整数\n");
    } else {
        printf("只写入了 %zu 个整数\n", count);
    }

    fclose(fp);
    return 0;
}
```

这里 `arr` 是数组名，本身就是地址，所以直接写 `arr`。`fwrite` 会把整个数组的内容（5 × 4 = 20 字节）写入文件。

---

## 4. 写结构体数据

`fwrite` 非常适合写入结构体，因为结构体在内存中也是连续的一块（尽管可能有内存对齐，但 `fwrite` 会原样写入）。

假设有一个 `struct Student` 数组，想写入文件：

```c
#include <stdio.h>

struct Student {
    char name[20];
    int age;
    float score;
};

int main() {
    FILE *fp = fopen("students.bin", "wb");
    if (fp == NULL) return 1;

    struct Student stu[2] = {
        {"Alice", 20, 90.5},
        {"Bob", 21, 85.0}
    };

    size_t count = fwrite(stu, sizeof(struct Student), 2, fp);
    if (count == 2) {
        printf("成功写入 2 个学生\n");
    } else {
        printf("只写入了 %zu 个学生\n", count);
    }

    fclose(fp);
    return 0;
}
```

之后你就可以用 `fread` 把这个文件读回来，恢复成结构体数组。

---

## 5. 返回值的重要性

**一定要检查返回值！** 如果返回值小于你请求的 `nmemb`，说明写入没有完全成功。可能的原因：
- 磁盘空间不足。
- 达到了文件大小限制（比如某些系统或平台限制）。
- 写入过程中发生错误（比如文件权限问题）。

如果发生错误，可以用 `ferror(fp)` 来检查是否出错。

```c
if (count < nmemb) {
    if (ferror(fp)) {
        printf("写入错误发生\n");
    }
}
```

---

## 6. 文件打开模式

- 如果是写入二进制数据，**强烈建议**使用 `"wb"`（写二进制）。这样数据会原样写入，不会做任何转换。
- 如果是写入文本文件，可以用 `"w"`（文本模式），但要注意文本模式下，某些系统会把换行符 `\n` 转换为 `\r\n`（比如 Windows）。这可能会导致二进制数据被破坏，所以写二进制数据务必用 `"wb"`。

---

## 7. 缓冲区刷新

`fwrite` 写入的数据通常会先进入**缓冲区**（内存中的一块区域），等缓冲区满了或者你调用 `fclose` / `fflush` 时，才会真正写入磁盘。如果你想确保数据立即写入磁盘（比如在嵌入式或日志系统中），可以在 `fwrite` 后调用 `fflush(fp)`。

```c
fwrite(...);
fflush(fp);  // 立即将缓冲区数据写入磁盘
```

不过一般情况下，`fclose` 时会自动刷新缓冲区，所以不需要特别操心。

---

## 8. 注意事项

### (1) 目标文件必须可写
确保文件打开成功，且你有写入权限。

### (2) 字节数计算正确
要确保 `size * nmemb` 是你要写入的数据总字节数。比如写 10 个 `double`，应该是 `sizeof(double) * 10`。写结构体数组，就是 `sizeof(struct X) * 个数`。

### (3) 不要写未初始化的数据
如果 `ptr` 指向的内存区域没有有效数据，写进去的就是乱码（垃圾值）。

### (4) 与 `fread` 的对称性
如果你用 `fwrite` 写了一个结构体，那么用 `fread` 读回来时，必须用同样的结构体定义、同样的顺序、同样的平台（注意内存对齐和字节序可能不同）。如果文件要在不同机器间交换，需要考虑字节序（大端/小端）和结构体对齐问题。

---

## 9. 常见错误和调试

- **忘记包含头文件**：`#include <stdio.h>`。
- **文件打开模式错误**：写二进制用了 `"w"` 而不是 `"wb"`，可能导致数据损坏（尤其在 Windows 上）。
- **忘记检查返回值**：没有检查实际写入个数，可能导致数据不完整而不自知。
- **指针传错**：比如传了指针的地址（`&ptr`）而不是指针本身，会把指针变量的值写入文件，而不是指针指向的数据。
- **缓冲区未刷新**：如果程序异常退出，缓冲区里的数据可能没写入文件。

---

## 10. 总结

- `fwrite(数据来源地址, 每个元素大小, 元素个数, 文件指针)`  
- 作用：**把内存中的数据写入文件**。  
- 常用于保存数组、结构体等二进制数据。  
- 必须用合适的模式打开文件（写二进制用 `"wb"`）。  
- 检查返回值，确保写入个数符合预期。  
- 与 `fread` 配合可以实现数据的持久化存储。
- 调用 `fflush(fp)` 可以立即将缓冲区数据写入磁盘。
